This week's structural theme is access control catching up with agentic AI: Anthropic proposed the first cross-industry severity scale for model jailbreaks, Jamf shipped OS-level governance for AI tools on the Mac fleet, and Google gated its new agent-to-cloud bridge behind IAM policy — three vendors converging on the same question of what an agent should be allowed to touch. Underneath that, the capability and cost layers kept moving: Alibaba tightened its video model's production quality, and DeepSeek set a deadline that will force real migration work on anyone still calling its legacy endpoints.

## <span class="beat">providers</span> Anthropic redeploys Claude Fable 5 with a proposed industry-wide jailbreak severity scale

Fable 5 and Mythos 5 came back online July 1 — Mythos 5 for US organizations only, approved June 26 — ending the 19-day global suspension that began June 12 under US export controls. The redeployment ships a retrained cybersecurity classifier that blocks the jailbreak technique Amazon researchers had reported in over 99% of cases; blocked requests aren't refused outright but rerouted to Claude Opus 4.8. A day later, Anthropic published a proposed Cyber Jailbreak Severity (CJS) scale developed with "Glasswing partners" Amazon, Microsoft, and Google: four axes — capability gain, breadth, ease of weaponization, discoverability — combining into a 0–4 severity rating, with a public HackerOne channel for submissions. This is the first attempt at a shared, cross-vendor yardstick for jailbreak severity rather than each lab grading its own incidents; if it gets adopted beyond the founding partners, it becomes a reference point for enterprise vendor-risk assessments of frontier model access. [Anthropic: redeploying Fable 5](https://www.anthropic.com/news/redeploying-fable-5) · [Anthropic: safeguards and jailbreak framework](https://www.anthropic.com/news/fable-safeguards-jailbreak-framework)

## <span class="beat">frameworks</span> Google Cloud opens a governed MCP bridge into its Agent Platform

On June 30, Google Cloud shipped a fully managed remote MCP server for Gemini Enterprise Agent Platform, giving external agents — built in Claude Code, Antigravity CLI, or any MCP-compliant tool — a standard way to call Model Garden models, pull shared prompt templates, manage notebooks, and hit model tuning and evaluation endpoints without leaving the IDE. Access runs through eight dedicated toolset endpoints and is gated by Cloud IAM Deny policies, with the Agent Registry acting as a searchable catalog of what's been exposed. The significant part is the framing, not the plumbing: MCP is moving from a developer convenience into an enterprise control plane, letting external tooling reach cloud AI infrastructure without an org giving up governance over which agent can touch which resource. [Google Cloud Blog](https://cloud.google.com/blog/products/ai-machine-learning/gemini-enterprise-agent-platform-remote-mcp-server/)

## <span class="beat">enterprise</span> Jamf brings native AI governance to the Mac fleet

Also June 30, Jamf shipped general availability of AI Governance inside Jamf for Mac — the first native, OS-level control plane for AI tool usage on managed Macs. It pairs a plain-language Policy Builder with fleet-wide visibility into which AI apps and MCP servers are actually running (including CLI-based coding agents and background processes, i.e., shadow AI), enforces policy through Declarative Device Management so users can't tamper with it, and generates audit-ready posture reports for CISOs. Day-one support covers Claude Code, Claude Desktop, and OpenAI Codex on AWS Bedrock, with Cursor and GitHub Copilot planned next. For any organization that has quietly accumulated developer-installed coding agents across its Mac fleet with zero visibility, this is the first tool that surfaces and governs that usage at the endpoint rather than bolting on network-level monitoring. [Jamf blog](https://www.jamf.com/blog/ai-governance-for-mac/)

## <span class="beat">vision</span> Alibaba's HappyHorse 1.1 tightens video-generation quality as rivals retreat

Alibaba released HappyHorse 1.1 on June 23, the first upgrade since April's limited beta: stronger interpretation of multiple reference images for reference-to-video consistency, smoother motion in complex action sequences, better instruction-following, and tighter audio-visual sync. It's live now at happyhorse.com and via API on Alibaba Cloud Model Studio; Alibaba's own post gives no pricing detail. The context that makes this matter: OpenAI discontinued Sora as financially unsustainable and ByteDance shelved Seedance 2.0's international rollout over Hollywood copyright complaints, leaving HappyHorse as one of the few enterprise-accessible video models still shipping a steady cadence of quality upgrades rather than retreating. [Alibaba Cloud blog](https://www.alibabacloud.com/blog/happyhorse-gets-stronger-motion-expressiveness-higher-generation-consistency-and-enhanced-visual-quality_603293)

## <span class="beat">providers</span> DeepSeek sets a hard deadline to kill its legacy V3-era API endpoints

DeepSeek's own API documentation confirms that `deepseek-chat` and `deepseek-reasoner` — the endpoint names tied to the V3/R1 generation — will be "fully retired and inaccessible" after July 24, 2026, 15:59 UTC, forcing every integration onto the V4 line (V4-Pro at 1.6T total/49B active parameters, V4-Flash at 284B total/13B active, both with 1M-token context and dual thinking/non-thinking modes). Unlike a soft deprecation notice, this is a fixed cutoff date with no stated grace period. Any team with model names hardcoded against the old endpoints needs a migration ticket before July 24 — and it's a reminder that low-cost API providers can sunset endpoints as abruptly as anyone else. [DeepSeek API docs](https://api-docs.deepseek.com/news/news260424)
